How this would work in production

Gemini already marks target language words with **bold markers**. Before sending text to ElevenLabs TTS, we could:

Alias approach: Maintain a small lookup table of known homographs per language. When we see a bold-marked word that matches the table, swap it for the phonetic alias. This is precise and only affects problem words.

language_code approach: Pass the target language code with every request. Simpler, but may cause English words to be mispronounced instead (the opposite problem).

Listen to both and decide which sounds more natural overall.
