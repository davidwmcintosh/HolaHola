# LinguaFlow - Interactive Language Tutor

## Overview
LinguaFlow is an AI-powered language learning application providing interactive conversation practice, vocabulary building, and grammar exercises in 9 target languages. It uses AI for personalized chat, flashcards, and grammar modules, adapting to user progress. The project aims to capture significant market potential in educational technology by supporting multiple languages and difficulty levels with explanations in any native language.

## User Preferences
Preferred communication style: Simple, everyday language.

## System Architecture

### UI/UX Decisions
The frontend features a mobile-first responsive design using Shadcn/ui (Radix UI base) with Tailwind CSS, adhering to Material Design principles. It includes light/dark mode, a two-column desktop layout, and a single-column mobile layout. Key UI elements include a prominent "Voice Learning" button with a "Recommended" badge, a subtle "Type instead" button, and a hint bar promoting voice learning. Typography uses Inter for UI text and JetBrains Mono for code/phonetic content. The application also supports PWA features, including offline caching, home screen installation, and auto-update notifications, with Capacitor configured for native iOS and Android builds sharing 100% of the web codebase.

### Technical Implementations
The frontend is built with React and TypeScript (Vite), utilizing Wouter for routing and the React Context API with TanStack Query for state management. The backend is an Express.js (Node.js) server with TypeScript, exposing a RESTful API. Data storage is managed via an abstract `IStorage` interface with Drizzle ORM for PostgreSQL. AI integration uses Gemini 2.5 Flash (via Replit's AI Integrations, billed to credits) for text chat completions. Session management uses `connect-pg-simple` with a PostgreSQL session store, and authentication is handled by Replit Auth (OIDC) supporting email/password and social logins, with WebSocket connections authenticated via server-side session validation. Stripe integration for subscription management is handled by `stripe-replit-sync`.

### Feature Specifications
LinguaFlow offers a conversational onboarding process, an adaptive multi-phase conversation system for gradual language immersion, and user agency allowing AI to suggest topics which require user confirmation. A production-stable REST-based voice pipeline (Deepgram Nova-3 STT → Gemini → Google Cloud TTS) supports push-to-talk recording with smart language handling: Deepgram auto-detects the user's spoken language (allowing questions in any language), while TTS uses the target learning language voice consistently to ensure authentic pronunciation of target language words even within English explanations. This "Split Response Architecture" delivers fast text-only responses with background enrichment for vocabulary and image generation. Content guardrails ensure appropriate learning content. Personalized learning features include scenario-based learning, slow pronunciation with phonetic breakdowns, automatic vocabulary extraction, spaced repetition, streak tracking, progress charts, and auto-difficulty adjustment. AI-generated educational images are intelligently displayed with caching. The application supports various subscription tiers (Free, Basic, Pro, Institutional) with differing features and AI model access, along with atomic voice message usage tracking. Both voice and text chat share a unified architecture for consistent instructions and context, with AI-generated conversation titles for easy retrieval and resumption.

**Voice Mode Enhancements (November 2025)**: The voice chat experience now includes three key improvements: (1) Auto-play greeting - when starting a new voice conversation, the AI's initial greeting is automatically spoken aloud with race-condition-safe state management using refs to prevent conflicts with active recordings; (2) Navigation shortcuts - "Call Tutor" (sidebar) and "Start Practicing" (hero) buttons now force creation of new conversations like the "New Chat" button, using accessible Link components with localStorage flags; (3) Keyboard accessibility - Enter key toggles microphone recording with proper guards to prevent interference with input fields, improving hands-free operation.

**Immersive Voice & Subtitle System (Difficulty-Adaptive Architecture)**: The voice chat provides an optimal learning experience with a strategic audio/visual split: (1) Voice speaks English explanations with authentic Spanish accent - the Spanish TTS voice (es-US-Chirp-HD-O) naturally pronounces both English AND Spanish words, creating immersive accent exposure without overwhelming learners; (2) **Adaptive Multi-Phrase Subtitle System** - When AI provides encouragement (e.g., "¡Perfecto!"), the Spanish encouragement word is **both spoken and shown**: TTS speaks "¡Perfecto!" with authentic pronunciation, followed by the English explanation. Subtitle behavior adapts based on difficulty level: **Beginner** - shows ALL teaching phrases in sequence (encouragement → phrase 1 → phrase 2 → ... → requested phrase) for comprehensive vocabulary exposure; **Intermediate** - limits to 4 intermediate phrases maximum to prevent overwhelm while maintaining vocabulary reinforcement; **Advanced** - skips intermediate phrases entirely, showing only encouragement → requested phrase for focused practice on complex sentences. For example, if the AI teaches greetings and says "Try saying 'Buenos días'!", a beginner sees "¡Perfecto!" → "Buenos días" → "buenas tardes" → "buenas noches" → "Buenos días" (persists), while an advanced learner sees only "¡Perfecto!" → "Buenos días" (persists). Server-side validation detects encouragement words, prepends them to TTS speech, extracts teaching phrases from the native field using Unicode-aware regex matching both straight quotes (\', \") and smart quotes (\u2018, \u2019, \u201C, \u201D, \u201A, \u201E), identifies the specific phrase the tutor asks the student to repeat (from "Try saying X" patterns with smart quote support), and applies difficulty-based limits via the `subtitlesJson` database field. Subtitle timer cleanup uses refs with proper useEffect cleanup to prevent stale updates during rapid message transitions. The final phrase **persists on screen after audio finishes**, providing a visual reference for what the student should practice; (3) AI responses use structured JSON format with "target" field (Spanish word/phrase for subtitle) and "native" field (English explanation with Spanish words embedded naturally). Questions ending with "?" are automatically removed and replaced with "Try saying it!"; (4) Full conversation history in the History view shows both languages for reference. This approach ensures learners at all levels hear clear explanations while seeing appropriately scoped target language vocabulary, maximizing comprehension and authentic accent exposure.

**ACTFL Proficiency Standards Integration**: LinguaFlow now tracks student proficiency using ACTFL World-Readiness Standards (Novice Low → Distinguished). The system maintains two independent dimensions: (1) difficulty level (beginner/intermediate/advanced) for content complexity, and (2) ACTFL proficiency level for demonstrated competency. The AI is ACTFL-aware across all 3 conversation phases and auto-tags messages with appropriate proficiency indicators. 

**Can-Do Statements Feature (November 2025)**: Complete implementation of ACTFL Can-Do Statements for self-assessment: (1) **Comprehensive Database** - 1,053 statements across 9 languages (Spanish, French, German, Italian, Portuguese, Japanese, Korean, Chinese, English) × 9 proficiency levels (Novice Low → Advanced High) × 3 communication modes (interpersonal, interpretive, presentational); (2) **Filterable API** - Endpoints at `/api/actfl/can-do-statements` with query params for language, level, and category filtering; (3) **Student Progress UI** - Self-assessment page at `/can-do-progress` with filterable cards per proficiency level, tabbed views by communication mode, checkbox tracking with stable content-based IDs (no state desynchronization across tabs), and progress summary metrics; (4) **AI Integration** - System prompts enhanced with 9 representative Can-Do statements (3 per category) aligned to learner's current ACTFL level, providing concrete teaching goals during conversations. Two critical bugs fixed during implementation: (a) checkbox state now tracked via React useState instead of constant Set, (b) stable statement IDs using content hashing prevent cross-tab state desynchronization. Backend persistence for Can-Do progress deferred to refinement phase. The integration supports institutional adoption by aligning with state curriculum standards while maintaining the existing 3-phase immersion system.

**Week 1 Quick Wins Package (November 2025)**: Three features leveraging Gemini's 1M context window to transform learning continuity:

**(1) Resume Conversations**: Context limits increased from 10/20 messages to 100/150 messages (voice/text modes), enabling full conversation history. When resuming conversations with >20 messages, the system automatically detects resumption and adds contextual prompts instructing the AI to: (a) Welcome students back warmly with acknowledgment of the time gap, (b) Reference specific past topics, words, and progress from conversation history, (c) Offer to continue previous topics or redirect to new content, giving students agency. A UI indicator displays "Resuming conversation · X messages · Last active [date]" at the top of both text chat and voice history views. This transforms conversations from isolated sessions into continuous learning journeys.

**(2) Smart Search**: Backend API (`/api/search/messages`) with storage layer enables case-insensitive search across all user conversations. SearchDialog component provides intuitive search interface with real-time results, conversation context, and click-to-navigate functionality. Students can search for specific topics, words, or phrases learned weeks ago and instantly jump to those conversations.

**(3) AI-Powered Practice Suggestions**: Endpoint (`/api/practice-suggestions`) analyzes the last 30 messages from recent conversations using Gemini to identify: (a) Common mistakes or confusion patterns, (b) Grammar/vocabulary gaps, (c) Topics not yet covered, (d) Areas where student shows readiness to advance. Returns 3-5 personalized practice recommendations (e.g., "You often confuse ser/estar - let's practice that!") with pattern analysis, actionable suggestions, and learning rationale. Enables proactive, data-driven practice recommendations.

### System Design Choices
Core data models include Users (with Stripe billing fields), Conversations, Messages, VocabularyWords, GrammarExercises, UserProgress, CulturalTips, and Topics, all linked via `userId`. Stripe data is synced to a PostgreSQL `stripe` schema via webhooks. User learning preferences such as `targetLanguage`, `nativeLanguage`, `difficultyLevel`, `actflLevel`, and `onboardingCompleted` are stored. The database schema includes ACTFL proficiency level tracking (`actflLevel` field) across Users, Messages, GrammarExercises, and UserProgress tables. Institutional features include CanDoStatements, StudentCanDoProgress, CurriculumPaths, TeacherClasses, and Assignments tables for state standards alignment. A PWA install prompt appears after 3 seconds, encouraging users to add the app to their home screen.

## External Dependencies

### Third-Party Services
-   **Stripe**: Payment processing and subscription management.
-   **Replit Auth**: OIDC authentication.
-   **Gemini API**: For text chat completions (via Replit AI Integrations, billed to credits). Uses Gemini 2.5 Flash (Free/Basic/Institutional) and Gemini 2.5 Pro (Pro tier).
-   **Deepgram API**: For voice STT using Nova-3 model with auto-detect mode - transcribes user's actual spoken language (English, Spanish, or any language).
-   **Google Cloud Text-to-Speech**: For authentic native pronunciation using Chirp 3 HD voices (Spanish, English) and Neural2 voices (French, German, Italian, Portuguese, Japanese, Korean). TTS uses target language voice mode: when teaching Spanish, all AI responses use Spanish voice for authentic pronunciation of Spanish words, even within English explanations.
-   **Unsplash**: Stock educational images.
-   **Gemini Flash-Image**: AI-generated contextual images (via Replit AI Integrations).

### Libraries & Tools
-   **Database**: Neon PostgreSQL, Drizzle ORM, Drizzle Kit.
-   **UI Framework**: React, TypeScript, Vite, Wouter.
-   **UI Components**: Radix UI, Shadcn/ui, Tailwind CSS.
-   **State Management**: TanStack Query, React Context.
-   **Billing**: `stripe-replit-sync`.
-   **Utilities**: `date-fns`, Embla Carousel, `franc-min` (language detection).

### AI Models
-   **Text Chat**: `gemini-2.5-flash` (Free/Basic/Institutional tiers), `gemini-2.5-pro` (Pro tier).
-   **Voice STT**: Deepgram `nova-3` with auto-detect mode - transcribes user's actual spoken language (English, Spanish, or any language). Achieves <300ms latency and 54.3% better WER than Whisper.
-   **Voice TTS**: Google Cloud voices in target language mode - uses the learning language voice for all AI responses (`es-US-Chirp-HD-O` for Spanish lessons, `en-US-Chirp-HD-O` for English lessons, Neural2 voices for other languages). Ensures authentic pronunciation of target language words.
